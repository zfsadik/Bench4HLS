You are given a four-bit input vector in. We want to know some relationships between each bit and its neighbour:

Implement a circuit in HLS that operates on a 4‑bit input vector in and produces three outputs as follows:

  (1) out_both (3 bits): Each bit of this output vector should indicate whether both the corresponding input bit and its neighbour to the left (higher index) are '1'. For example, out_both[2] should indicate if in[2] and in[3] are both 1. Since in[3] has no neighbour to the left, the answer is obvious so we don't need to know out_both[3].

  (2) out_any (3 bits): Each bit of this output vector should indicate whether any of the corresponding input bit and its neighbour to the left (higher index) are '1'. For example, out_any[2] should indicate if either in[2] or in[3] are 1. Since in[3] has no neighbour to the left, the answer is obvious so we don't need to know out_any[3].

  (3) out_different: Each bit of this output vector should indicate whether the corresponding input bit is different from its neighbour to the left. For example, out_different[2] should indicate if in[2] is different from in[3]. For this part, treat the vector as wrapping around, so in[3]'s neighbour to the left is in[0].

The top-level function should have the following prototype:
void TopModule(ap_uint<4> in, 
               ap_uint<3> &out_both, 
               ap_uint<3> &out_any, 
               ap_uint<4> &out_different)
